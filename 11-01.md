### Задание 1. СУБД

### Кейс
Крупная строительная компания, которая также занимается проектированием и девелопментом, решила создать 
правильную архитектуру для работы с данными. Ниже представлены задачи, которые необходимо решить для
каждой предметной области. 

Какие типы СУБД, на ваш взгляд, лучше всего подойдут для решения этих задач и почему? 
 
1.1. Бюджетирование проектов с дальнейшим формированием финансовых аналитических отчётов и прогнозирования рисков.
СУБД должна гарантировать целостность и чёткую структуру данных.

### Ответ:

Для такой задачи подойдет реляционная СУБД.

1.2. Под каждый девелоперский проект создаётся отдельный лендинг, и все данные по лидам стекаются в CRM к 
маркетологам и менеджерам по продажам. Какой тип СУБД лучше использовать для лендингов и для CRM? 
СУБД должны быть гибкими и быстрыми.

### Ответ:

Для такой задачи подойдет реляционная/графовая СУБД.

1.3. Отдел контроля качества решил создать базу по корпоративным нормам и правилам, обучающему материалу 
и так далее, сформированную согласно структуре компании. СУБД должна иметь простую и понятную структуру.

### Ответ:

Для такой задачи подойдет документно-ориентированная СУБД.

1.4. Департамент логистики нуждается в решении задач по быстрому формированию маршрутов доставки материалов 
по объектам и распределению курьеров по маршрутам с доставкой документов. СУБД должна уметь быстро работать
со связями.

### Ответ:

Для такой задачи оптимальным решением будет использование графовой СУБД, т.к. она предназначена для работы со связями, графами, узлами и свойствами.

### Задание 3. SQL vs NoSQL

3.1. Напишите пять преимуществ SQL-систем по отношению к NoSQL. 

### Ответ:

1. Масштабируемость - вертикальная масштабируемость против горизонтальной, т.е. при необходимости расширения, достаточно будет усовершенствовать один сервер - увеличить объем свободных ресурсов сервера (ЦП,ОЗУ хранилища) 
2. Стандартизация - стандартизированный язык минимализирует риски при работе с БД.
3. Согласованность - базы данных SQL имеют обеспечивают согласованность данных в большом количестве таблиц за счет использования транзакций и гарантируют, что изменения внесенные в одну таблицу отобразяться во всех связанных таблицах.
4. Безопасность - БД SQL имеют встроенные средства безопасности, которые помогают защитить данные от нестанкционированного доступа (аутетификация пользователя, контроль доступа, шифрование)
5. Резервирование - БД имеют встроенные средства резервного копирования и восстановления, которые помогают в случае системных сбоев.

здесь справится лучше всего и почему?

*Приведите ответ в свободной форме.*

Для такой задачи я бы оринетировался на решение Hadoop, т.к. эта система позволяет работать с большими объемами данных, использует кластер из множества серверов, что обеспечивает отказоустойчивость и масштабируемость.
